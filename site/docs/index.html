Get started

Inspect before you manage.

From install to a complete local report in about one minute. The first two commands are read-only.

01 / Install

Install agentctl

The installer supports macOS and Linux, requires Node.js 20 or newer, installs without sudo, and verifies the downloaded release checksum.

curl -fsSL https://agentctl.justrepl.com/install.sh | bash

Review the installer source before running it.

PATH recovery

If ~/.local/bin is not on your PATH, the installer prints the exact export line to add. It does not edit shell startup files.

02 / Observe

Run a read-only inspection

agentctl inspect

agentctl reads known configuration roots, hashes SKILL.md definitions, inventories skill script directories, and inspects configuration key names. It does not execute any target CLI, plugin, hook, skill, or script.

Inspect one target

agentctl inspect --target codex
agentctl inspect --target claude

Inspect a fixture or alternate home

agentctl inspect --home /path/to/home

03 / Interpret

Read the report

Inventory
Detected roots and counts for skills, agents, commands, rules, guidance, config files, scripts, hooks, and MCP surfaces.
Duplicate group
Two or more skills with the same SHA-256 digest for their SKILL.md content.
Divergent name
The same skill directory name resolves to different SKILL.md digests across targets.
Potential secret field
A key name resembles a credential-bearing field. Only the key and line number enter the report; the value does not.
Script surface
A regular file exists under a skill's scripts/ directory. Presence is not proof that it ran.

04 / Automate

Use stable JSON

agentctl inspect --json > agent-environment-report.json

JSON output contains no ANSI formatting and is versioned as agentctl.inspect/v1alpha1. The schema is published in the repository.

Fail a policy check on potential secrets

agentctl inspect --strict --json

--strict exits with status 5 when potential secret-bearing fields are found. A successful inspection with ordinary informational findings exits 0.

05 / Diagnose

Check the local runtime

agentctl doctor
agentctl doctor --json

Doctor checks the Node.js version, platform, home readability, Git availability, and target roots. It remains read-only.

06 / Opt in

Initialize canonical source

agentctl init ./my-agent-environment --dry-run
agentctl init ./my-agent-environment

init is an explicit write command. It creates a new Agent Environment as Code repository and refuses to overwrite a non-empty directory. Target-native rendering is not part of v0.1.

07 / Recover

Uninstall cleanly

The installer adds only a versioned directory and one command symlink. It never modifies agent configuration or shell startup files.

rm "$HOME/.local/bin/agentctl"
rm -r "$HOME/.local/share/agentctl"

If you used AGENTCTL_INSTALL_DIR or AGENTCTL_LIB_DIR, remove the corresponding explicit paths instead.

Release boundary

What v0.1 does not claim

  • Filesystem presence does not prove a target loaded a resource.
  • Matching files do not prove equivalent behavior across models.
  • Key-name detection does not prove a value is a valid credential.
  • agentctl does not yet import, render, apply, reconcile, or roll back target configuration.

These are explicit roadmap capabilities, not implied current behavior. Follow development on GitHub.
