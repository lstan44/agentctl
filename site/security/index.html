Security model · v0.1

Trust is a behavior,
not a badge.

agentctl begins with the narrowest useful authority: read known configuration roots, report evidence locally, and change nothing.

01

No execution

Inspection never invokes target CLIs, skills, scripts, hooks, MCP servers, or plugins.

02

No writes

inspect and doctor never alter agent roots, project files, or shell configuration.

03

No exfiltration

The CLI has no telemetry or upload path. Potential secret values are excluded from report objects.

Inspected assets

What the inspector reads

Known global configuration roots for supported targets, selected top-level configuration files, SKILL.md content for hashing, and regular file metadata under skill scripts/ directories.

Symbolic links are not traversed during recursive resource walks. Configuration files larger than 2 MiB are skipped and marked in the report.

Secret boundary

Keys may be reported. Values may not.

The inspector searches configuration lines for credential-like key names such as API keys, access tokens, authorization fields, client secrets, passwords, and private keys. It records the key name and line number only.

Installer

Versioned, verified, no sudo

The public installer downloads a versioned archive and its SHA-256 file from the GitHub release, verifies the archive, installs into a staged version directory, then atomically changes the active symlink. It does not edit your PATH or shell files.

A checksum from the same release channel detects corruption but is not an independent signature. Signed build provenance is a v1 release gate.

Mutation boundary

Writes require a different command.

agentctl init is explicit and separate from inspection. It writes only to a newly selected or empty directory and refuses to overwrite existing files. Later reconcile operations will require content-addressed plans, ownership records, staging, atomic replacement, and tested rollback before release.

Disclosure

Report vulnerabilities privately.

Use GitHub private vulnerability reporting for findings involving secret exposure, execution, path escape, configuration corruption, installer compromise, or release integrity.

Do not place exploit details or real configuration in a public issue.
